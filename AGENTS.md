# 運用ルール

## ドキュメント
- チームで共有すべきストック情報（要件定義、設計ドキュメント等）は `docs/` ディレクトリに配置し、Git で管理する。

## 設計プラン
- プランファイルは `docs/plans` ディレクトリに保存する。
- プランが確定したら、実装に入る前に以下を必ず行う:
  1. プランファイルを連番プレフィックス + 内容がわかる名前にリネームする（例: `001-database-schema-design.md`）
  2. 下記の参考資料セクションの索引に追加する
  3. 新しいプランにより `docs/` 内の既存ドキュメントに古い記述が生じた場合は更新する
  4. プランファイルにタスクリスト（チェックボックス形式）のセクションを作成する
  5. 上記の変更をまとめてコミットする
- 実装中はプランファイルのタスクリストに沿って作業し、タスク完了時にチェックを入れてコミットする。これにより未完了の作業をいつでもプランファイルから把握できる。
- 実装中に方針の変更が生じた場合はプランファイルも更新する。

## Git 操作
- タスクが一つ完了したタイミングにコミットする。
- タスクの完了判定は開発者による動作確認を要することが多いので、その場合コミットは開発者の確認を取ってから行う。
  - ただし開発者から自律的に進めるよう指示された場合はこの限りではない。

## MCP ツール
- **xcode** — プロジェクトのビルドができる。実装後はビルドが通るか確認すること。
